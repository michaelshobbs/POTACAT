# The Rig Swap with POTACAT Cloud Foundation

**Headline:** Two pieces of new infrastructure that change what POTACAT can be — **POTACAT Cloud Tunnel** lets the mobile app reach your rig from anywhere without Tailscale or port forwarding, and **Guest Pass** lets you hand your rig to a friend for a fixed window with real guardrails. Plus a fully refactored JTCAT state machine with a 152-case regression suite, a massive SSTV decoder rewrite, WWBOTA bunker spots, and a long tail of audio / reliability fixes.

## POTACAT Cloud Tunnel — one-tap remote

Reach your rig from anywhere on the public internet. No Tailscale, no port forwarding, no router permission slip from your spouse.

POTACAT spawns `cloudflared` as a background child process and opens an outbound tunnel to Cloudflare. The mobile app connects over `https://<callsign>.cloud.potacat.com`. Setup is one button on the Cloud tab — sign in with your POTACAT Cloud account, hit **Enable POTACAT Cloud**, copy the pairing QR.

- **Live state indicator** in the system tray and on the ECHOCAT settings panel: *LAN only* / *Provisioning…* / *Live* / *Reconnecting…* / *Error*.
- **Health-check loop** restarts the cloudflared child if it stops emitting tunnel-up events; tray icon reflects the in-flight state without polling.
- **Pairing QR** auto-embeds the cloud hostname when the tunnel is live, falls back to LAN otherwise. The same QR, different host — phone doesn't care.
- **Per-platform bundled binaries** ship inside the installer (Windows .exe, macOS arm64, Linux x64). No extra install step for the user.
- **HTTP/2 control plane** — cloudflared is pinned to `--protocol http2` instead of QUIC's default. QUIC was tearing down its control stream every ~30s under sustained renderer load (visible as "control stream encountered a failure while serving" + reconnect spirals). HTTP/2 trades ~10–30 ms of stream latency for a connection that survives GC pauses and CPU jitter. WebRTC audio doesn't traverse the tunnel anyway, so audio latency is unaffected.

The Tailscale path still works — there's an *alternative VPN* section on the ECHOCAT tab with the same MagicDNS + HTTPS-cert instructions that used to be primary. For users who already have Tailscale running on every device, nothing changes.

## Guest Pass — hand the rig to a friend with guardrails

You can now generate a time-boxed, privilege-limited access token for someone else to operate your rig over ECHOCAT. Out-of-band tunes blocked, TX power clamped, session auto-ends at expiry, guest logs to their own logbook.

- **Open Settings → ECHOCAT → Sharing & Recovery → Guest Pass**, set start/end, license class, max TX power, station call, operator call, optional control op. Hit *Generate pass* — POTACAT mints a code + share URL + QR.
- **Desktop enforcement layer** — every CAT command from the paired guest passes through a pass-state interceptor before reaching the rig. Tunes outside license-class band privileges are dropped before they hit the rig CAT. TX power is clamped to whatever you set.
- **Owner banner** at the top of the desktop while a pass is active — shows the guest's call and time remaining. One-click *Revoke* if you need to end it early.
- **License class → band privileges** mapped per the FCC 97.301 table (Tech, General, Extra).
- **§97.119 ID compliance is yours to manage** — the form lets you decide how `STATION_CALLSIGN`, `OPERATOR`, and the control operator are filled. POTACAT records what you set; no hard-coded legal stance.
- **Revoke broadcast** — pass revocation propagates to the connected mobile client immediately over the existing WebSocket; their session closes without a phone restart.
- **SmartSDR / Flex gating** — Flex-specific commands (slice freq, atu, etc.) honor the same band/power gates the generic CAT interceptor enforces, no second code path to maintain.

## JTCAT — extracted state machine, 152-case regression suite, compressed-reply support

The JTCAT QSO state machine moves out of `main.js` into `lib/jtcat-state-machine.js` as a pure function with `engine` + `log` injected. A 152-case unit-test suite (`test/jtcat-test.js`) covers every phase transition for both CQ-mode and reply-mode, the decode-classifier (`inferReplyStep`), the pre-encode race condition, and the WSJT-X parity spot checks. Runs in <2 s with no live radio. CI gates every push + PR.

Three documented WSJT-X departures encoded in the tests (don't try to "fix" them later):
- We log at reports-exchanged (not after RR73 courtesy), so a dropped courtesy doesn't lose the entry.
- `waiting` phase replaces hard abort when the station you called picks up someone else — holds for 12 cycles for them to free up.
- Slash-portable calls indexed both as-typed *and* by bare home call, for spot-matching robustness.

**New in 1.8.0: compressed-reply support.** Some operators auto-sequence past the grid step and reply to a CQ with a direct signal report (`K3SBP KB2ELA -12`). The state machine used to ignore these as "no advance — matched MYCALL but didn't parse `<call> <grid>`." It now recognizes the pattern, captures their report of us, computes our SNR-based report of them, and TX's `R+report` immediately — jumping straight to the courtesy cycle and logging on advance.

**Popout TX-message race fix.** The "stale — re-encoding for X" path was occasionally racing setTxFreq + setTxMessage and sending the wrong message at slot boundary. Pre-encode race is now provably ordered and tested against the real `lib/ft8-engine.js`.

**VITA-49 audio worklet.** Replaces the per-frame `createBuffer` + `createBufferSource` pattern in the renderer (which was allocating ~190 Web Audio nodes/sec on SmartSDR Direct) with a single `AudioWorkletProcessor` that owns a ring buffer + linear-interp resampler. Frame handler just `port.postMessage`s the Float32Array PCM. Eliminates the renderer-falling-behind spiral that drove the "Audio backpressure: N frames dropped" log lines and indirectly destabilized the cloudflared tunnel.

**Silenced false-positive log.** The "feedAudio all zeros — audio capture may be dead" diagnostic now skips during active TX. The rig's RX path is intentionally muted while we're keying — those zeros aren't a bug.

## SSTV decoder — MMSSTV techniques, 15 modes, regression-tested

Big rewrite of the SSTV decoder. Started as a Martin-1 quality push, ended up as a near-complete reimplementation that ports MMSSTV's approach to slant correction, post-processing, and noise robustness.

- **15 modes in the TX dropdown** — Martin 1/2/M2/M3/M4, Scottie 1/2/DX, Robot 24/36/72, PD 90/120/160/180/240.
- **PD-mode decoders** — PD90/120/160/180/240, plus a YCbCr-aware replay path that skips the per-line replay for these (full-buffer two-pass + slant correction handles them).
- **Robot 36 noise-pattern sensitivity** — +9.5 dB at 30 dB SNR seed=1. Re-tuned `syncBias` after the slant-correction landed.
- **MAD-based slant regressor** — robust against single-cycle outliers (clicks, mics, lightning) where the prior linear regressor jumped. Ratchet-quality baselines locked in.
- **Manual slant slider** — for the cases where the regressor can't lock in (very weak signals). Operator can nudge the slope by hand.
- **Two-pass full-buffer receive** — MMSSTV's approach. First pass establishes drift, second pass uses the corrected timing for pixel placement.
- **MMSSTV-style post-processing** — unsharp + saturation + gamma applied after decode. The default tuning is reversible if you don't like it (in Settings → SSTV).
- **129-case regression suite** + **49-cell PSNR quality matrix** + **24 pending feature contracts** — `test/sstv-test.js` and `test/sstv-quality-test.js` are CI-gated.
- **Non-SSTV audio false-emit guard** — quality matrix now includes a "should not emit" cell for music, voice, and noise inputs. Catches accidental "decoded a song" regressions.
- **dax_tx direct → tx-state broadcast** — when SSTV finishes TX over the direct VITA-49 path, the desktop pushes a tx-state-back event so the mobile clears its "transmitting" banner immediately.

Per-mode quality grades from the 2026-05-31 end-of-session push: Martin **A**, Scottie **A−**, Robot **B+**, PD **D+** (PD is in flight — the YCbCr 4:2:0 path needs more work).

## WWBOTA — Worldwide Bunkers on the Air spot source

New first-class spot source covering UKBOTA / HBBOTA / USBOTA / ITABOTA / FBOTA / ONBOTA and the rest of the WWBOTA family. Filed as Issue #34 by BexiBuxi.

- **Live spots** from `api.wwbota.net/spots/?age=1`, refreshed every cycle alongside POTA / SOTA / WWFF / LLOTA / Tiles. Default **on**.
- **N-fer aware** — bunker activations regularly tag 5+ references at once. POTACAT shows the primary reference compactly (`B/ON-1047 +4 more`) and keeps the secondary refs available for the log dialog.
- **Re-spot from QSO log** — unauthenticated POST to the WWBOTA cluster (no token needed per their OpenAPI), with the bunker reference auto-prepended to the comment per their requirement.
- **Slate-gray row tint + WWBOTA badge** — distinct from the existing palette (POTA green, SOTA orange, WWFF teal, LLOTA blue, Tiles purple).
- **QRT filter compatibility** — WWBOTA's `type:"QRT"` is appended to the comments field so the existing "Hide QRT spots" filter catches them.
- **Cross-program X-Ref** — the activator panel's cross-program reference panel adds a WWBOTA row, alongside SOTA / WWFF / LLOTA.

A mobile-side handoff is filed at `docs/ios-handoffs/wwbota-spot-source.md` for the iOS / Android app to pick up the new source shape.

## ECHOCAT — always-on, panel redesign, Cloud Tunnel as the primary path

The ECHOCAT mobile-pairing system has matured enough that the *Enable ECHOCAT* checkbox stops earning its keep — it's now always-on.

The whole Settings → ECHOCAT tab is reorganized around the Cloud Tunnel as the recommended path:

1. **POTACAT Cloud Tunnel** banner at the top, reflecting live tunnel state from the Cloud tab.
2. **Pair Mobile App** — the QR generator, paired-device list.
3. **Tailscale (alternative VPN)** — promoted out of its old collapsible nested under Pair into its own section.
4. **Remote CW Keyer** — own section, including the optional DTR-keyed CW Key Port for non-Flex rigs.
5. **Connection / Audio & PTT / Sharing & Recovery / Advanced** — purposeful groups instead of the old wall of flat checkboxes.

The pair state machine is **cloud-first**: when you have a POTACAT Cloud account + the tunnel is live, the QR points at the cloud hostname; if not, it falls back to LAN (or Tailscale if MagicDNS is up). No more guessing what the QR resolves to.

The legacy Remote Launcher checkbox (the duplicate of the rich install/uninstall block) is removed. One canonical launcher control, in the Sharing & Recovery section.

## Settings search — row-level filter

Searching for "CW" in Settings used to mark the entire **Tuning** fieldset as a match because *CW XIT Offset* lives there next to *Scan Dwell Time*, then show the whole fieldset including unrelated rows. Now the search walks each fieldset row by row:

- Containers (`<div>`, `<details>`, `<section>`, etc.) recurse into children — children that don't match get hidden.
- Atomic rows (labels, help-text spans, paragraphs) show or hide as a unit. A `<label>CW Key Port: <input></label>` stays together with its value.
- `<option>` text inside hidden `<select>` controls is no longer considered "visible." This was why **Net Reminders** used to surface for "CW" — the mode select in its hidden editor has a `<option>CW</option>`.
- Application-hidden subtrees (`.hidden` class, `[hidden]` attr, inline `display:none`) are skipped both as match candidates and as filter targets.
- Section heading containers stay whole: when the heading text matches, the entire body shows. So "CW" in the Macros tab brings the full CW Macros editor along with the heading, not just the heading row.
- `<summary>` elements are preserved so `<details>` blocks never lose their label.

## ECHOCAT internals

- **In-process spectrum FFT** gated on mobile subscribe — the desktop only spends the FFT cycles when someone's actually subscribed, rather than running unconditionally.
- **Auth-deferred kick** — the post-connect cleanup that disconnects stale paired devices now waits for the new client to finish auth, with enriched payload + any-msg liveness.
- **Cloud Tunnel: kill stale cloudflared on tunnel rotation** — if the Cloud account flips tunnel IDs (rare, but possible during account migrations), the old cloudflared child is reaped before the new one spawns.
- **Settings dialog: null-stub legacy launcher refs** — was throwing a TypeError on dialog open after the duplicate launcher UI was removed; now safely no-ops.
- **remote-server: stamp paired-device lastSeen even in no-token mode** — fixes the paired-devices list showing all entries as "never connected" for users who run ECHOCAT without an access token.

## Reliability

- **Cloud Tunnel auto-recovery** — 5-min health check loop + state manager scaffold; if cloudflared's logs go silent for too long, it gets restarted.
- **JTCAT TX message race** — final fix landed (`99bdd21`), backed by regression tests.
- **JTCAT popout reply on stale QSO state** — fixed; consecutive fresh-reply clicks now consistently advance the state machine.
- **Audio backpressure** in non-live windows — the JTCAT popout's audio path no longer accumulates IPC backlog when it's not the active consumer.
- **Flex CWX hardware sidetone mute** for macro duration — eliminates the doubled-sidetone artifact when the WinKeyer and the rig both emit sidetones during a macro send.
