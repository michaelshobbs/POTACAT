# POTACAT on the Raspberry Pi

POTACAT now ships an **arm64 Linux build** — so it runs natively on the Raspberry Pi, the sBitx, and other ARM single-board computers.

## arm64 Linux builds

Until now, POTACAT's Linux packages were x86_64 only. On an ARM board — a Pi 4, a Pi 5, an sBitx — the AppImage failed before it even started with `Exec format error`, because the binary was the wrong CPU architecture.

This release adds native arm64 (aarch64) packages alongside the existing x86_64 ones:

- `POTACAT-1.5.24-arm64.AppImage`
- `potacat_1.5.24_arm64.deb`
- `potacat-1.5.24.aarch64.rpm`

Grab the `-arm64` / `aarch64` file for a Raspberry Pi or sBitx; the existing files (no arch suffix, or `amd64` / `x86_64`) are still the ones for a normal PC. Auto-update is wired up for arm64 too, so future releases land automatically.

Note: this is a **64-bit** build. Modern Raspberry Pi OS is 64-bit by default — if you're on an older 32-bit image, you'll need to reimage to 64-bit to run POTACAT.

---

**Tailscale users on macOS:** if Settings → ECHOCAT says "Tailscale not detected" but you have it installed, enable MagicDNS in your [admin DNS console](https://login.tailscale.com/admin/dns). The status line tells you the exact toggle and links straight to the page.

**macOS users on Apple Silicon:** the DMG is ad-hoc signed (no paid Developer ID yet). If you see "POTACAT is damaged and can't be opened," run this once from Terminal and relaunch:

```
sudo xattr -dr com.apple.quarantine /Applications/POTACAT.app
```
