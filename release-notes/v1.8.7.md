# Desktop-as-a-Client over LAN, Tailscale and Cloud

**Headline:** v1.8.7 makes POTACAT-as-a-client real. Pair your laptop to your shack desktop and drive the rig from anywhere — over your LAN, your tailnet, or the POTACAT Cloud Tunnel — and redeem a friend's **Share My Rig Guest Pass right on the desktop**, no phone required. Plus a deep FT8/JTCAT correctness pass that fixes the most-reported sequencing bugs.

## 💻 Desktop as a Client

- **➕ Add a remote radio — one paste box, any credential.** More → Remote Radios now has a labeled add row that accepts a `potacat://pair` link, a Guest Pass share link, **or just the 4-word pass code**. Redemption happens in-process — no OS protocol-handler registration needed, which also fixes pasted pair links doing nothing on dev builds and **Linux AppImages**.
- **Guest Pass on the desktop.** The full guest flow that previously existed only in the ECHOCAT mobile app: paste a pass (or click the landing page's button — `potacat://pass/…` now works on desktop), POTACAT redeems it (signed-in or anonymous), auto-connects to the owner's shack, and shows a 🎟 badge with the pass's privilege class, power cap, and live expiry. When the session ends, you're switched back to your local rig automatically with a clear notice. QSOs you make land in **your** logbook with the host's call as STATION_CALLSIGN (§97.119).
- **Remote QSY fixed** — tune requests from a client desktop arrived 1000× too small ("7.026 kHz") and were refused; the unit mismatch is fixed and the wire contract is now regression-tested.
- **Pair links lead with the LAN address** — on Tailscale-equipped desktops, pairing QRs/links put the MagicDNS name in the primary host field, which a not-yet-tailnetted phone can't resolve. First-time pairing now works. — **HI3NLER**
- **Cloud device registry fixed** — desktop registration silently crashed (`_ensureValidToken is not a function`), which blocked cloud-account auto-pair end to end.
- **Clearer auth diagnostics** — every ECHOCAT auth attempt now logs which credential it presented, so "invalid token" mysteries are debuggable from the CAT log.

## 📻 JTCAT / FT8 correctness pass

- **Replies to your CQ get a report, not your grid.** The classic bug: someone answers `CQ K3SBP FN20` with `K3SBP A1BCD FN30`, you double-click, and POTACAT sent `A1BCD K3SBP FN20` (as if they were CQing). Root causes fixed: the popout's cached callsign could go stale, and the "addressed to me" match broke on portable/hashed renderings of your own call. The reply step is now re-derived authoritatively from the raw decode text.
- **Non-standard CQs parse correctly.** Directed and event CQs with no grid (`CQ NA K1ABC`, `CQ POTA W1AW`, `CQ TEST K1ABC`) and numeric-serial contest CQs (`CQ 075 K1ABC FN42`) used to pick the modifier as the callsign — you'd transmit `NA K3SBP FN20`. One shared, tested parser now backs the popout, the main window, the phone, and the maps.
- **Clock-sync is real now.** The old "Sync: OK" lit on every cycle regardless of your clock. JTCAT now measures your PC's offset against NTP and raises a banner when FT8 decoding is doomed (with one-click **Time settings** / **Sync now** / **Recheck**). And when your clock is corrected mid-session, the engine **re-baselines live** — decodes resume without restarting POTACAT.
- **TX audio interleave fixed** — with the ECHOCAT audio bridge connected, an FT8 transmission interleaved the (silent) phone-mic stream into the same DAX TX stream: garbled on-air signal and dead RX decodes for the next cycle. The mic path is now gated to voice TX only.
- **FT4 mode is remembered** across restarts (it used to silently revert to FT8 and park you on the wrong sub-band), and FT2's RX window now matches its cycle length exactly.

## ⚙️ Settings polish

- **Save is always visible** — the Save/Cancel bar is pinned at the bottom of the Settings dialog at every window size; no more scrolling past the footer to save.
- **Cloud tab reorganized** — POTACAT Cloud sign-in is first (sign-in needs only email + password now; the callsign field appears only when creating an account), then POTACAT Tunnel, Logbook Sync, Embeddable Widgets, Backup & Recovery, and Sign Out at the bottom. POTA.app Profile moved to the Station tab where it belongs. Also fixed the "Cloud QSOs" counter flickering.
- **RBN + PSKReporter Propagation default ON** for new installs, so the "where am I being heard" view has data out of the box.

## Under the hood

- 290+ automated regression tests now gate the FT8 parser, QSO sequencing, clock re-baseline, and the new guest/remote auth paths.

**macOS users on Apple Silicon:** the DMG is ad-hoc signed. If you see "POTACAT is damaged and can't be opened," run this once and relaunch:

```
sudo xattr -dr com.apple.quarantine /Applications/POTACAT.app
```

(In-app auto-update is disabled on macOS — grab the new DMG from the GitHub release directly.)
