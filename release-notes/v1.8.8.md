# macOS hotfix — app wouldn't launch since v1.8.6

**Headline:** if POTACAT on your Mac has been launching and instantly quitting — no window, no error — since v1.8.6, this release fixes it. Thanks to **N3VD** for the Terminal trace that pinned it down.

## 🍎 The fix

Since v1.8.6, POTACAT advertises itself for phone/laptop discovery on every real network interface. On macOS, that discovery socket collided with the system's own Bonjour service (`mDNSResponder`) on port 5353, and the resulting error killed POTACAT **before the window ever opened** — silently, because it happened underneath the crash-dialog handler. The "app is damaged" message some users saw was an unrelated red herring (the usual unsigned-app quarantine), which is why the `xattr` fix didn't help.

- macOS now shares port 5353 with `mDNSResponder` the way the OS expects — so network discovery actually **works** on Mac now, instead of crashing the app.
- Discovery errors of any kind can no longer take POTACAT down, on any platform — worst case, discovery is disabled on one network interface and a note lands in the CAT log.
- Windows and Linux discovery behavior is unchanged.

## 📻 Also fixed: QSY off FT8 now stops the engine

Tuning a CW or SSB spot (including from the phone) while JTCAT was running used to leave the FT8/FT4 engine decoding garbage on the new frequency, with the mobile FT8 tab stuck on "Stop". QSYing to a non-data mode now stops the engine cleanly; JTCAT's own band changes are unaffected.

## 🔎 Also new: startup.log

Every launch now writes a `startup.log` into POTACAT's data folder (`~/Library/Application Support/POTACAT/` on macOS, `%APPDATA%\potacat\` on Windows) with timing checkpoints and full stack traces for any fatal error — so if the app ever fails to open again, that one file tells us exactly why. If you report a "won't start" issue, attach it.

## Upgrading on macOS

Your installed 1.8.6/1.8.7 can't auto-update (it can't launch), so download the v1.8.8 DMG from this release directly. The DMG is ad-hoc signed — if you see "POTACAT is damaged and can't be opened," run this once and relaunch:

```
sudo xattr -dr com.apple.quarantine /Applications/POTACAT.app
```

Windows users get this release through the normal auto-update.
