# Massive bug fixes, Cloud Sharing, and even more bug fixes

**Headline:** v1.8.6 rolls up everything since v1.8.4 into one big release — the new **Cloud Sharing** stack (pair a laptop or a friend's station to your rig, with tap-to-pair and away-from-home fallback) plus a long list of user-reported bug fixes across CAT control, logging, audio, and pairing. Thanks to everyone who sent reports — most of the fixes below have your callsign on them.

## ☁️ Cloud Sharing

The big one. POTACAT can now act as a **client of another POTACAT** — run it on a laptop and drive the rig that's connected to your shack desktop, or let a friend operate your station.

- **Desktop-to-desktop pairing** — a laptop dials your shack desktop over a 3-leg chain (LAN → Tailscale → POTACAT Cloud), with cert-fingerprint pinning. Add stations under **More → Remote Radios**; a banner + status chip show when you're driving a remote rig.
- **Share My Rig** — hand your station to someone else with a share link. Choose **start-now or scheduled** (date/time pickers), optionally allow **anonymous** guests, and the pass list shows the guest's station callsign + a Copy-Link button (no 4-word code on screen for shoulder-surfers). Trust tiers: owned links never expire; guest links cap at 30 days.
- **Tap-to-pair** — on your home LAN you no longer need a QR scan. Tap your desktop in the ECHOCAT app, click **Approve** on the popup that appears on the desktop, done. 60-second auto-deny, OS notification when POTACAT is minimized, and a toggle under Settings → ECHOCAT → Advanced → Connection (defaults on; Cloud-Tunnel exposure always blocks it).
- **Tailscale-host propagation** — a paired phone automatically learns your tailnet hostname and falls back to it when you leave home. Pair once at the house; Tailscale "just works" away from home with no re-pair.
- **Cloud-account auto-pair** — devices signed into the same POTACAT Cloud account can authorize each other without a QR.

## 🐞 Bug fixes — thank you for the reports

- **FT-710 / FT-991A / FT-891 power-on reliability** — the power-on button was ~50% from the desktop and never worked from ECHOCAT mobile. These rigs park their CAT UART asleep in standby and drop the first wake byte; POTACAT now sends `PS1;` and a second `PS1;` 500 ms later. — **KB2UXB**
- **Voice macros recorded as a "series of tones"** — recording grabbed the OS-default input, which on most POTACAT setups is the rig's USB CODEC, so it captured TX monitor audio. Added an explicit **Mic Input picker** at the top of the Voice Macros editor. — **John WZ1H**
- **JTCAT "RX low — bump gain" hint** — a new pre-gain tap detects "signal present but gain too low" and shows a pill next to the gain slider, so a default-quiet RX path doesn't cost you days. — **chris KF0JFW**
- **FT8/FT4 QSOs from mobile silently corrupted** — the phone's blank RST defaulted to `59`; it's now mode-aware (`-00` for FT8/FT4/JS8/JT65/JT9).
- **FTX-1 Field / Optima** — follow-up rig controls, corrected power scaling, model-prefixed power commands, native Field tuner command, and an Optima antenna selector. — **Hitman90210 (#42)**
- **Logbook "+ New QSO" auto-fills Frequency** from the rig's current frequency (manual edits still win). — **N4DWJ**
- **Flex per-band antenna mapping** — map RX/TX antennas per band on the rig profile; POTACAT switches antennas before the frequency lands. — **KO6M**
- **External ATU / Tune docs** — clarified that the "Auto-tune ATU on band change" toggle is the rig's *internal* ATU, and pointed RF-sensing-tuner (LDG/MFJ) users at the FT8 Tune button / Custom CAT Command. — **Scott**
- **WS message-size logging** now lands in the user-visible CAT log (was console-only) with a new >8 MiB tier — easier to diagnose oversized pushes. — **Walt KK4DF**
- **WinKeyer "Invalid handle" error on every quit** — a disconnect race where `port.close()` beat the in-flight write; disconnect now waits for the write callback.
- **WebSDR crash** — a `pcmArr` ReferenceError took the whole main process down the first time WebSDR audio arrived with an ECHOCAT WS client attached. Fixed (and the ECHOCAT branch restored to boxed-array form so the browser decoder can read it).
- **mDNS "Found on your network" came up empty** on machines with Hyper-V / WSL / Docker / Tailscale — POTACAT was publishing on one OS-chosen (often virtual) interface. Now publishes on every real LAN interface.
- **Tap-to-pair popup vanished after a few seconds** because iOS retires idle HTTP sockets aggressively — the popout now lives its full 60 s regardless of the phone's socket state, and Approve still works on a dead socket (the phone picks up credentials on retry).
- **`STATION_CALLSIGN` is now written on every saved QSO** — LOTW and contest-log validation both want it, and it matters for §97.119 attribution when a remote operator drives the rig.
- **WWBOTA spot window** widened 1 h → 3 h (the 1 h window often came up empty from US time zones).
- **"Hide" column toggle did nothing** — the column-visibility CSS selector was missing.
- **Report a Bug** now closes the Settings dialog so the recording banner is actually visible.
- **Remote Launcher install path** — five interlocking bugs that together blocked iOS reach: `launcher.js` was missing from the packaged app, no way to start it without an uninstall/install dance, a missing-script case needing the same dance, a multi-op callsign-path regression, and an HTTP-only status probe that misreported HTTPS launchers as dead. Plus a launcher log file for future diagnosis.

## ✨ Also in this release

- **Charcoal dark theme** — a neutral, OLED-friendly alternate dark palette. Pick Navy or Charcoal under Settings → Display.
- **Contests view redesign** — replaced the 14-card grid with a single "What's Next" list bucketed by time (Live now / This weekend / This week / …), plus a per-source multi-select.
- **Settings dialog** — sidebar layout at every width, footer reorg with an ECHOCAT mobile promo, and the old toolbar Settings dropdown rehomed into the sidebar + Summary → Quick Settings.
- **Logbook polish** — color-coded columns (Date/Time/Callsign/Park, etc.) that stay readable across light and dark themes, and **Ctrl + / Ctrl −** zoom in the Logbook window (syncs to the main table size until you set your own).
- **Architecture B** — the host now forwards client-driven QSOs to a paired desktop or guest-pass session instead of always logging them locally.

---

**macOS users on Apple Silicon:** the DMG is ad-hoc signed. If you see "POTACAT is damaged and can't be opened," run this once and relaunch:

```
sudo xattr -dr com.apple.quarantine /Applications/POTACAT.app
```

(In-app auto-update is disabled on macOS — grab the new DMG from the GitHub release directly.)
