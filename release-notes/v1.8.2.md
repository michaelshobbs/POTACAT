# The Cloud Tunnel Release (fixes)

**Headline:** Same release as v1.8.1 — a redesigned Settings dashboard, multi-operator profiles, Cloud Tunnel self-healing, and the Share Your Rig dialog — but with three critical bug fixes found within the first hour of v1.8.1 going live. v1.8.1 was never publicly distributed; v1.8.2 is the build you actually want.

## Fixes since v1.8.1

- **Pairing via Cloud Tunnel hostname** — the `_handleHttpRequest` gate that closes off the renderer SPA when the tunnel is exposed was returning 503 for `/api/pair` too, which is exactly what the mobile app uses to redeem the pairing token. AB9AI hit this immediately on first pair attempt. `/api/pair` is now explicitly whitelisted (already protected by a single-use 32-byte hex token with a 5-minute TTL). `/api/ptt/*` deliberately stays gated — it's an unauthenticated local-trust shortcut for iOS Shortcuts and must not be reachable over the tunnel.
- **`ReferenceError: req is not defined`** in `_handleMessage` — the Club Mode removal left the auth-message branch referencing the HTTP upgrade request, but the method signature only took `(ws, msg)`. Crashed POTACAT every time a Cloud-Tunnel or token-auth client sent its auth frame.
- **EADDRINUSE on port 7300** — when a previous Electron process orphaned the port (Windows TIME_WAIT, force-kill, fast restart), the server emitted `'error'` with no listener on the main side, which `EventEmitter` throws as uncaughtException and killed the whole app. ECHOCAT now retries listen 5× with 800ms backoff, then logs softly without re-emitting `'error'` so a port conflict doesn't take POTACAT down.

## What shipped in v1.8.1

All the headline features below are part of v1.8.2 — v1.8.1 was pulled before meaningful adoption.

## Settings, redesigned

Settings has grown into a real dashboard. The dialog itself is now responsive — drag the corner to resize and POTACAT remembers your size, automatically lays out as a sidebar at ≥720px and stacks everything at narrower widths, and switches into a compact mode on short screens so the Save button never gets pushed off the bottom.

The new **Summary** tab opens by default and lives at the top of the dialog. Five cards give you the at-a-glance state of POTACAT:

- **Operator** — your callsign + grid, plus the dropdown for switching between operators (see below).
- **Logbook** — total / SSB / CW / FT8/4/2 / SSTV QSO counts pulled live from your ADIF, with one-click Open / Import / Export.
- **POTACAT Cloud** — Active / Free Trial / Inactive pill, a Sync now button that surfaces real server errors instead of swallowing them, and an honest "Manage subscription" link that goes to BuyMeACoffee.
- **ECHOCAT** — tunnel state pill (Cloud Tunnel / Tailscale / LAN only), your paired devices with rename + revoke, Pair / Share / Turn off tunnel / Diagnostics buttons.
- **POTA.app** — Hunter parks · QSOs · awards · endorsements and Activator activations · parks · QSOs, refresh on demand.

Operator + Logbook share a row at default width and collapse to a single column on narrow dialogs.

Below the cards is a new **Quick settings** collapsible. Today it holds CW Keyer (enable, type, sidetone + pitch + volume). Collapsed by default so it stays out of the way for the majority of operators who don't use it; the full keyer setup still lives under Station → CW Keyer.

## Multi-operator profiles

POTACAT can now host more than one operator per install. Useful for households where two hams share a shack PC, club stations with a roster, and anyone who wants a separate callsign + logbook + macros for special-event ops.

- **Operator dropdown** in the Summary's Operator card switches the active operator with one click. POTACAT restarts to apply.
- **Each operator gets their own**: callsign, grid, watchlist, ADIF logbook path, macros, QRZ / Wavelog / HAMRS / Cloudlog / POTACAT Cloud credentials, scan dwell, CW XIT.
- **Rigs[] stays shared** — three operators sharing a Flex 6700 don't each carry their own copy of the same hardware list.
- **Storage** is `userData/profiles/<callsign>/settings.json` with a tiny global `settings.json` for machine-scoped state. Migration runs automatically the first time v1.8.2 starts — your existing settings become a profile named after your `myCallsign`. Idempotent on subsequent boots.
- **Add operator** from the Summary card prompts for a callsign and seeds a fresh profile with its own logbook path.
- **Manage Operators** dialog lists profiles with an Archive action. Archive moves the folder to `profiles/_archived/<callsign>-<timestamp>/` — reversible by hand if you change your mind.

## Share Your Rig (Guest Pass owner UI)

Hand the rig to a friend for a fixed window, with real guardrails. The previous Guest Pass form was buried inside ECHOCAT settings and led with a QR code. v1.8.2 promotes it to a dedicated dialog launched straight from the Summary's ECHOCAT card, and rebuilds it around how operators actually use it — most will email the link, not stand next to their friend with a phone.

- **Duration presets**: 1 hour / 4 hours / 1 day / 7 days, plus a custom numeric input that caps at 720 hours (30 days). 1 day is the new default.
- **License privileges** dropdown groups US (Technician / General / Extra) plus Canadian (Basic / Honours) plus an Unrestricted / unmetered option. Selecting a class shows the band scope so you know what your friend can do.
- **Max TX power** presets at 5 / 25 / 100 / 500 / 1500 W. Rigs that can't report TX power over CAT get TX disabled instead as a failsafe.
- **§97.119 callsign fields** for station, operator, and control op. The station + control op fields auto-fill from your callsign; the eCFR link to the actual rule text is one click away.
- **Result view leads with the share link**, not the QR. A big Copy link button, plus "Email it…" that pops your default mail client with a pre-filled subject and body containing the link, expiry, and privilege summary. The QR is in a collapsible — opt-in for in-person scans.
- **My recent passes** lists everything you've issued, with per-row revoke and a "guest connected now" badge for active sessions.
- **Network-reach notice** at the top of the form tells the truth about who can reach your rig right now: LAN-only / Tailscale / Cloud Tunnel. When you're not on Cloud Tunnel you get a friendly nudge that explains the limitation and offers a Join POTACAT Cloud link.

## POTACAT Cloud Tunnel — orphan adoption, diagnostics, real errors

The Cloud Tunnel infrastructure that shipped in v1.8.0 was solid in the happy path but brittle when things drifted between Cloudflare's records and POTACAT's database. v1.8.2 makes it self-healing.

- **Orphan tunnel adoption** — if Cloudflare already has a tunnel under your account's deterministic name (from a previous provision that crashed mid-flow, a manual cleanup, anything), provision now finds it and adopts it into the database instead of trying to create a duplicate and bouncing off CF error 1013. Symptom this fixes: "Turn on tunnel" silently returning HTTP 500 with no actionable feedback.
- **Idempotent DNS step** — if a CNAME record already exists at `<callsign>.potacat.com`, the provision flow now updates it in place rather than crashing on the duplicate POST.
- **Verbose 5xx errors** — when the cloud returns a 500 with a JSON `{ error, message }` body, the desktop now surfaces the actual reason instead of the generic "POTACAT cloud is error" line. Drastically narrows the time-to-diagnose on the next intermittent failure.
- **Diagnostics button** on the Summary's ECHOCAT card opens a live dump of your tunnel state: DB row contents, CF tunnel record existence + status, CF DNS record existence + content, and whether the cloud server's Cloudflare API env is intact. Copy-to-clipboard for raw JSON if you ever need to share it.
- **Correct lifecycle pills** — the tunnel state pill now correctly shows Provisioning… → Connecting… → Cloud Tunnel as cloudflared registers each connection. Was previously stuck on LAN only because of a field name mismatch (`ts.state` vs `ts.status`).
- **Tailscale detection fixed** — when the tunnel is off but you're signed into a tailnet, the pill now correctly reads "Tailscale" instead of "LAN only". Same root cause as the Cloud Tunnel pill bug.

## ECHOCAT card + paired devices

The ECHOCAT card on Summary surfaces what used to live across two settings tabs:

- **Paired device list** — per device: name, paired date, "last seen" age. Rename inline (Electron blocks `window.prompt`, so the rename action opens a proper modal). Revoke with confirmation.
- **Share Your Rig…** button launches the new Guest Pass dialog (cloud-backed, privilege-gated). Distinct from **Pair my device…** which still opens the existing pair popout for your own iPhone / iPad with short-TTL device tokens.
- **Turn on/off Cloud Tunnel** reflects user-intent state, not the momentary status — so the button doesn't flicker mid-startup.
- **Diagnostics** opens the new tunnel-diagnostics dialog described above.
- The same card is mirrored on the ECHOCAT settings tab, so configuration and control sit next to each other.

## ECHOCAT settings tab, decluttered

The ECHOCAT tab in Settings has been reorganized around the new Summary cards:

- Cloud + ECHOCAT cards at the top (mirrors of Summary).
- **Remote CW Keyer** — enable + CW Key Port.
- **Audio & PTT** — PTT safety timeout, SSB-over-DATA, Restart audio bridge, TX EQ + Compression.
- **Remote Launcher** — its own section now.
- **Tailscale (alternative VPN)** — collapsed by default with the install steps and current tailnet status inside. Most users don't need to see it; it's there when you do.
- **Advanced** — port, access token, STUN, pair host override, TLS cert/key paths, force-renew Tailscale cert. All the knobs nobody touches until they have to.

Gone from this tab: the duplicate Cloud Tunnel status banner, the standalone Pair Mobile App section, the inline Guest Pass form (the new dialog supersedes it), the old "Sharing & Recovery" wrapper.

## Club Station Mode removed

The Club Station Mode CSV-of-credentials feature has been retired. It overlapped completely with the more secure paired-device tokens (mobile pairing) and Guest Passes (friend share with privilege caps), without the operational baggage of plaintext passwords, manual hashing, and a CSV schema nobody could remember. Anyone who needs the multi-operator-at-the-same-rig use case is better served by the new Guest Pass flow. ~600 lines of code, two dedicated lib files, and one settings section gone.

## Polish

- **What's New dialog** — actually renders markdown now. The literal word "Headline:" no longer appears at the top of release notes; the lead paragraph is styled distinctly; sections get real headings with subtle separators; inline code, code blocks, italics, and links all render properly.
- **Dialog → external link routing** — any anchor with `data-external` anywhere in the renderer now routes to your default browser via a single document-level capture-phase handler. Was previously per-element and missed some bindings.
- **Custom prompt dialog** — replaces calls to `window.prompt()`, which Electron renderers silently block. Used by Add Operator, Rename Device, and any future flows that need a string from the user.
- **Sync now button** in the Cloud card disables itself while a sync is in flight and surfaces server errors directly. No more wondering whether the click registered.
- **Settings dialog → Operator card** "Manage…" link opens a small modal listing your operators with archive actions and the active operator clearly marked.

## Cloud server side (deployed)

These changes were deployed to `api.potacat.com` ahead of this desktop release. Listed for completeness:

- `GET /v1/cloud-tunnel/diagnostics` — verbose state dump consumed by the new desktop Diagnostics button.
- `POST /v1/cloud-tunnel/provision` — gained orphan-tunnel adoption (the CF 1013 fix) and idempotent DNS handling.
