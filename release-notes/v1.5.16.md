Headline: the iOS POTACAT app pairs reliably now. Tailscale's free Let's Encrypt certs are baked into the pair flow — click "Open pairing QR" and POTACAT does the rest.

## ECHOCAT — iOS pairing actually works

After a long debug session with K3SBP and K0OTC, we cracked why iOS pairs were hitting "network request failed" with no useful diagnostics on either side. Two compounding issues:

1. **iOS App Transport Security rejects self-signed certs.** The current iOS app build sets `NSAllowsArbitraryLoads: true` AND `NSAllowsLocalNetworking: true`. Apple's docs say when both are present, the arbitrary-loads flag is ignored — ATS gets enforced normally for non-local-network destinations, killing the connection at the TLS trust evaluation before any app code runs. No SAN tweak, no fingerprint pinning workaround, nothing on the desktop side could fix it.
2. **Our cert generator made several wrong assumptions** about which IPs and hostnames to embed in the SAN, and the IP picker was hijacked by Hyper-V virtual switches and USB tethering interfaces on Windows machines, embedding addresses that were unreachable from the user's phone.

The fix is **a real publicly-trusted cert via Tailscale**, fully automated:

- **One-click pair flow.** Click "Open pairing QR." If you've got Tailscale running with MagicDNS + HTTPS Certificates enabled, POTACAT issues a Let's Encrypt cert via `tailscale cert <hostname>` (one short shell-out), reloads the HTTPS server with the new cert, then renders the QR. Subsequent pairs hit the cached cert and skip setup entirely.
- **Auto-renewal.** On every POTACAT startup, if the cached cert has fewer than 14 days left, POTACAT silently renews. No manual action ever — the cert just stays valid.
- **Precise diagnostics.** Settings → ECHOCAT shows the exact next step when something's missing: install Tailscale, sign in, enable MagicDNS, enable HTTPS Certs — each branch with a deep link to the right Tailscale admin page.
- **Per-platform Tailscale CLI lookup.** macOS users no longer need to run "Install Tailscale CLI" from the menu-bar app — POTACAT finds the binary inside `/Applications/Tailscale.app/Contents/MacOS/Tailscale` directly. Homebrew on Apple Silicon (`/opt/homebrew/bin/tailscale`) and Windows (`C:\Program Files\Tailscale\tailscale.exe`) are also probed.
- **Pair QR uses the Tailscale MagicDNS hostname** when Tailscale is up — `wss://shack.your-tailnet.ts.net:7300` rather than a brittle IP. Stable across IP changes; works for pairing AND roaming.
- **Advanced disclosure** in Settings → ECHOCAT exposes the power-user knobs without cluttering the default panel: pair host override, manual TLS cert/key paths (for users with corporate CAs, mkcert, or reverse proxies), and a force-renew button.

## ECHOCAT — Pair-QR fixes

- **Virtual / phantom adapters filtered from the IP list.** Hyper-V's Default Switch (vEthernet on `172.x`), WSL2's `vEthernet (Default Switch)`, and "Ethernet 6"-style USB-NCM phantom interfaces (real Intel MAC, no default route, unreachable) were polluting the Settings → ECHOCAT IP list and getting picked into the QR. Three-layer filter now: interface name patterns, MAC OUI checks (Hyper-V `00:15:5d`, VMware `00:50:56`, VirtualBox `08:00:27`, etc.), and on Windows, a default-route check (probed via `route print -4 0.0.0.0` — 50ms, cached).
- **DNS hostnames in cert SAN.** The self-signed fallback now includes the system hostname (and its `.local` variant) plus the Tailscale MagicDNS name in the SAN, regenerating the cached cert on startup if any current name or IP is missing. Belt-and-suspenders for the rare paths that still hit the self-signed path.
- **Three preloads no longer missing from packaged installs.** v1.5.14's `app.asar` was missing `preload-pair-popout.js`, `preload-prop-popout.js`, and `preload-log-popout.js` because they weren't enumerated in the `electron-builder` files manifest. Anyone running from source had working popouts; anyone on the installer got `window.api` undefined and silently broken windows. (Fixed in v1.5.15 but worth restating — this was the bug behind several "the QR popout shows blank fields" reports.)

## ECHOCAT — Logbook fixes

Two related bugs from Mallory KD5ZZU's report — both fixed in this release:

- **iOS "Log QSO" no longer silently drops the QSO.** The desktop's `log-qso` WebSocket handler was reading from `msg.data`, but the iOS app sends the payload flat at the top level. `msg.data` was undefined, the handler bailed with "Missing callsign", and the iOS UI showed haptic-success because `LogQuickSheet` doesn't wait for the result reply. Now accepts both shapes — browser nested `{type, data}` and iOS flat `{type, callsign, …}` — without breaking the existing browser path.
- **Desktop logbook changes now push to mobile in real-time.** `saveQsoRecord` (called from desktop UI, ECHOCAT logging, WSJT-X bridge, etc.) now broadcasts the updated full QSO list to all connected mobile/browser clients on every successful save. Previously the mobile view was stuck on the snapshot from connect-time and only refreshed on reconnect. The iOS app already subscribes to `all-qsos` so this lights up its logbook view automatically.

## CI

- **Windows build pinned to `windows-2022`.** GitHub flipped `windows-latest` to Windows Server 2025 between v1.5.14 and v1.5.15, and the new image's Visual Studio layout broke node-gyp's rebuild of `@serialport/bindings-cpp` with "Could not find any Visual Studio installation to use." Pinned explicitly so tag-pushes produce installers again.

---

**Tailscale users on macOS:** if Settings → ECHOCAT says "Tailscale not detected" but you have it installed, you may need to enable MagicDNS in your [admin DNS console](https://login.tailscale.com/admin/dns). The status line now tells you the exact toggle to flip and links straight to the page.

**macOS users on Apple Silicon:** the DMG is ad-hoc signed (no paid Developer ID yet). If you see "POTACAT is damaged and can't be opened," run this once from Terminal and then relaunch:

```
sudo xattr -dr com.apple.quarantine /Applications/POTACAT.app
```
