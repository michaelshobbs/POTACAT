# Rotor-EZ support, a friendlier JTCAT, and the Linux sandbox done right

**Headline:** v1.8.9 adds direct serial control of **Rotor-EZ / DCU-1 rotator controllers**, teaches JTCAT to **keep calling a busy station** the way WSJT-X does, and ends the **"chrome-sandbox must be setuid root"** saga on Linux — plus a crash fix every remote-control user will want.

## 📡 Rotor-EZ / DCU-1 rotor control

POTACAT can now drive Idiom Press / Ham Supply **Rotor-EZ** and **RotorCard** boards — and genuine Hy-Gain **DCU-1** controllers — directly over their serial port, as an alternative to PstRotator. Pick "Rotor-EZ / DCU-1 (serial)" under Settings → Tuning → Rotor control and choose the COM port; auto-rotate-on-QSY, the click-the-bearing column, and the ECHOCAT rotor toggle all work exactly as they do with PstRotator. The driver understands the controller's quirks: a new heading sent mid-rotation would silently cancel, so rapid QSYs are queued and dispatched once the rotor stops and the brake sets. — requested by **Jonathan KM4CFT**

## 📻 JTCAT: keep calling a busy station

When the station you called answers someone else, JTCAT used to park your QSO with a "⏸ waiting to reply" banner and disable TX. That's gone — it now does what WSJT-X does: **keeps calling** (tail-ending), bounded by the per-QSO tries setting so it can't run away. We also adopted WSJT-X's "auto stop to avoid accidental QRM" guard: if the DX answers someone else within 50 Hz of *your* TX offset, JTCAT halts rather than transmit on top of their QSO. Verified line-by-line against the WSJT-X source.

## 🐧 Linux: no more setuid-root sandbox

Running POTACAT on Linux no longer requires (or installs) a setuid-root `chrome-sandbox` (#37 — thanks **mshappe** for pushing on this):

- **.deb installs** now ship an AppArmor profile (the same mechanism Chrome and VS Code use), so POTACAT runs with Chromium's full user-namespace sandbox and **zero setuid binaries** — including on Ubuntu 24.04 with its new userns restrictions.
- **AppImage and hardened-kernel systems**, where the namespace sandbox is impossible, used to abort with a cryptic "make chrome-sandbox 4755" error before the app even started. A new launcher detects exactly that situation and starts POTACAT with the sandbox disabled instead, logging what happened and the better fixes. Setuid remains available as a manual, documented opt-in only.
- All three scenarios are now verified in CI on a real Ubuntu 24.04 runner on every validation build.

## 🐞 Bug fixes — thank you for the reports

- **Crash loop after connecting to an unreachable remote shack** — clicking Connect on a paired desktop that couldn't be reached crashed POTACAT, and because the connection persisted, it crashed again seconds after every relaunch. The transport error now surfaces in the log and the dial chain retries with backoff, as designed. — **Michael KE4EST**
- **Multi-state parks wrote garbage into the log's STATE field** — hunting a park that spans states (locationDesc like `US-WI,US-MI`) logged "WI,US-MI". Single-state parks fill STATE as before; multi-state parks now **ask you which state the activator is in** (or leave it blank). Phone-logged QSOs leave it blank rather than guess. — **Scott WG9I**
- **Revoking a paired device now actually disconnects it** — Revoke used to only delete the pairing record, so an already-connected phone kept full rig control until it reconnected. The live session is now kicked instantly with a proper "revoked" message (new protocol close code 4004), and a desktop driving a remote shack falls back to its local rig with a clear notice when its pairing is revoked.
- **Cloud backup download hardened** — an expired session no longer leaves the button dead (it refreshes and retries), an interrupted download can no longer leave a silently truncated ADIF at your chosen path (downloads go to a temp file and rename on success), and a deleted/expired account now prompts a re-sign-in instead of a cryptic error. The button is now "Download Cloud Backup (.adi)" with a dated default filename.
- **Tap-to-pair from the same LAN works while Cloud Tunnel is exposed** — being reachable from the internet no longer blocks pairing a phone that's sitting on your home network.
- **Settings → Cloud copy fix** — the tunnel address is `<callsign>.potacat.com` (the old `.cloud.` form lingered in the blurb and docs).
