SmartSDR Direct subscribes to Flex audio over UDP and bypasses the DAX program — useful when DAX gets stuck silent under RDP.
Advanced POTACAT tunes the radio directly over the API. This routes tuning through SmartSDR-Win's Kenwood CAT shim (TCP 5002–5005) instead — a fallback only, and SmartSDR must be running.
Connect to any radio with TCP CAT — Flex on a custom port, or other radios that speak raw Kenwood-style ASCII over TCP with no authentication. For an Elecraft K4 over the network use "K4 Remote (network)" below instead — the K4 uses a different framed + authenticated protocol.
Reach an Elecraft K4 / K4D over the network using the radio's built-in remote protocol. Set a remote password in the K4's menu first. Port 9205 = plain TCP (password sent as SHA-384 hash). Port 9204 = TLS-PSK encrypted (recommended for connections over the public internet); password is used as the pre-shared key.
Direct Kenwood-protocol CAT over serial. Recommended for QRPLabs QMX/QDX, and works with most radios that respond to FA/MD commands (Kenwood, Elecraft, etc.). Required for QRPLabs QMX/QDX (uses DTR for PTT). Also check for Digirig, SignaLink, and similar USB interfaces.
Direct CI-V binary protocol over USB. For IC-7300, IC-705, and other Icom radios with USB port.
For CW paddle keying via ECHOCAT, set your radio menu: SET > Connectors > USB SEND/Keying > USB Keying (CW) = DTR
Raw CI-V frames over TCP. Works with (a) the IC-7300 MK II / IC-705 MK II / IC-7610 MK II native Network CI-V over LAN, or (b) a ser2net / socat bridge on a radio-adjacent box that exposes a USB CI-V port as a TCP listener.
IC-7300 MK II default port is 50001. For ser2net, whatever TCP port you configured in /etc/ser2net.yaml. CW paddle keying works the same as direct USB CI-V — the codec sends the same CW key command (0x1C 0x01) over the TCP transport.
RS-BA1 protocol over UDP. Connect to (a) wfserver (wfview's headless server) running on a Linux/Mac/Pi adjacent to a USB-attached Icom — typical setup for IC-7300 / IC-7100 / FT-710 — or (b) directly to an IP-native Icom (IC-705, IC-9700, IC-7610, IC-7851, IC-R8600). Username/password come from whatever you set in wfserver's --setup, or in the radio's own Network User menu. Phase 1: CAT only (frequency, mode, CW text). No audio yet — bridge audio separately if needed (Mumble, virtual cable). For wfserver setup, see WFSERVER.md in the wfview repo.
Prevents PTT keying on Digirig, SignaLink, and similar interfaces Default 4532. Change if another app is using that port. For DigiRig / SignaLink / similar interfaces that key PTT via DTR or RTS.
Two-port adapters (full-size DigiRig): pick the SECOND COM port (the one without CI-V data); usually DTR.
Single-port adapters (DigiRig Mobile): pick the SAME port as CAT above; pin depends on the model — try DTR first, switch to RTS if PTT doesn't key.
Leave blank for rigs that PTT over CAT (most modern Yaesu / Icom).
